# Sample Descriptions

This folder provides **selected End-to-End Samples** from PrZMA.

## Directory Overview

### 1. All Execution Mode
Representative outputs from a full PrZMA execution workflow for schema tracking.

- `discord_full_trigger/`  
  Example configuration files used for full execution. In this example, Discord was used as the target platform.  
  (The related database analysis is presented on the main project page.)

- `run_...\`  
  Main execution output directory.
  Each run folder contains files such as `schema_tracking_discord_ft_1`, `schema_tracking_discord_ft_2`, and `schema_tracking_discord_ft_3`.
  - `snapshots/`  
    Snapshot result folders generated during the run.
  - `actions.jsonl`  
    Step-by-step action logs recorded during execution.
  - `actions.summary.json`  
    Summarized version of the recorded actions.
  - `schema_tracking_ ... .db`
    Schema-tracked database derived from cache dumps, used for structured artifact analysis across snapshots.
- `vm_snap/`
    Example PrZMA output showing browser-side web-state files captured at snapshot time during Telegram Web all-execution mode, including page HTML, DOM, and IndexedDB schema.


### 2. Education
Simplified example outputs from the education-oriented scenario.  
**This sample corresponds to the outputs shown in the demo video.**
The configuration files used here are available on the main project page.

- `snapshots/`  
  Snapshot result folders from the scenario.
- `actions.jsonl`  
  Action logs for the education workflow.
- `actions.summary.json`  
  Summary of the recorded actions.

### 3. Tool Validation
Example outputs used for tool-oriented validation and demonstration.  
In this example, **ChromeCacheView** is used as the target tool.

- `Output_ChromeCacheView_...`  
  Sample outputs exported from the external forensic tool.

- `TMI_ChromeCacheView_...`  
  Outputs generated by the Tool Manual Interpreter (TMI) after reading the tool specification.  
  In this example, the tool specification is provided in both **TXT** and **URL** formats.  
  Based on the target tool specification, TMI selects relevant artifacts from the artifact catalog and generates the configuration file (`przma_config.json`) for tool testing, which was manually prepared in the Education workflow.
  In this case, the Automation Agent is configured as a single agent, whose persona is defined to perform artifact-generating actions for tool testing.

### 4. Replay Mode
Example action logs showing both the original education run and its replayed execution generated by `endToEndReplay.py`
`endToEndReplay.py` is not LLM-based; it just replays an `actions.jsonl` file.

- `PrZMA_Education/actions.jsonl`: **Action Logs(actions.jsonl)** of original education run 
- `PrZMA_Education_Replay_[run_num]/replayed_actions.jsonl`: **Replayed Action Logs** generated from `PrZMA_Education/actions.jsonl` by `endToEndReplay.py`
